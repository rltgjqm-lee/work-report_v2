---
description: 현재 변경사항을 보고 커밋 메시지만 추천 (커밋은 실행하지 않음)
---

현재 변경사항에 대한 커밋 메시지를 추천해줘. **절대 `git add`나 `git commit`을 실행하지 마 — 메시지만 제안하고 끝낸다.**

1. `git status`로 어떤 파일이 바뀌었는지 확인한다.
2. staged된 변경사항이 있으면 `git diff --staged`를, 없으면 `git diff`를 봐서 실제 내용을 파악한다.
3. `git log --oneline -15`로 이 저장소의 최근 커밋 스타일을 참고한다.

이 저장소의 커밋 메시지 컨벤션:

- 첫 줄: `type: 한글 요약` (`feat`, `fix`, `refactor`, `docs`, `chore` 중 하나, 콜론 뒤 한 칸 띄우고 설명)
- 변경이 한 가지 초점이면 첫 줄만으로 충분
- 변경이 여러 갈래거나 "왜"를 설명할 필요가 있으면, 빈 줄 하나 두고 `-` bullet(또는 짧은 문단)으로 부연 — "무엇을 바꿨는지"보다 "왜 바꿨는지"에 무게를 둔다
- 한글로 작성, 코드/식별자명은 원문 그대로(백틱 없이) 인라인으로 섞어 써도 됨

출력은 추천 커밋 메시지 하나만 코드블록으로 보여주고, 왜 이 메시지를 골랐는지 한두 문장으로 짧게 덧붙인다. 여러 옵션을 나열하지 말고 가장 적합한 것 하나로 좁혀서 제안한다.
